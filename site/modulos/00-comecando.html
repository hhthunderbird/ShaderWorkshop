Comece aqui: o Playground

Antes de pintar pixels, 2 minutos pra conhecer a ferramenta. Você vai brincar com um quadradinho de cor — sem medo de quebrar nada.

Mexa sem medo

Troque os números do vec3 (são Vermelho, Verde, Azul, de 0 a 1) e clique ▶ Test Drive. Erre à vontade: ↺ Reset volta tudo, 💡 Mostrar solução mostra uma resposta.

Os botões

  • ▶ Test Drive — roda o que você escreveu na hora.
  • ↺ Reset — volta o código ao original, como se nada tivesse acontecido.
  • ✓ Conferir — aparece quando o exercício tem um alvo: compara seu resultado com ele.
  • 💡 Mostrar solução — mostra uma resposta pronta pra você comparar.
  • 🔁 Ver em HLSL — mostra como ficaria no Unity — ilustrativo, sem precisar entender agora.
Tela preta/cinza ou ⚠ ERROR? Quase sempre é erro de digitação — faltou um ; no fim da linha ou um parêntese. ↺ Reset restaura o código certo; 💡 Mostrar solução mostra a resposta. Não entre em pânico: errar é parte do jogo.

Onde eu escrevo?

Você só mexe na parte clara, entre as marcas // >>> EDIT e // <<< EDIT. O resto (mais apagado) é o motor — ele cuida da parte chata pra você.

🔤 Lendo uma linha de código pela 1ª vez (opcional)

Não precisa decorar — é só pra reconhecer quando ler:

  • void main() { ... } — a receita que a GPU roda pra cada pixel.
  • ; — fim de uma instrução (como o ponto final de uma frase).
  • gl_FragColor — a cor que sai (o resultado final do pixel).
  • vec3(r, g, b) — um trio de números (uma cor: vermelho, verde, azul).
  • float — um número (com vírgula, tipo 0.5).
  • step e mix — duas funçõezinhas que você usa já no Módulo 1 (degrau e mistura) — a gente explica direitinho lá.
Preciso saber programar?
Não! O curso ensina do zero. Aqui você só troca números e vê o que muda.
E se eu apagar algo importante?
↺ Reset traz tudo de volta. Nada quebra de verdade.
